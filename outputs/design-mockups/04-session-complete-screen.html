Design Improvement #4

Session Complete Screen

endSession() injects bare <h2>, <p>, <button> into .content which has text-align: left. The most important moment in the app currently looks like mid-exercise content.

Before
Left-aligned, unstyled — no sense of completion

Session Complete!

You completed session 1. Ready for session 2?

Finishing a session looks identical to a mid-exercise card. The heading sits left-aligned, the button has no class. There's zero visual celebration for completing the session.
// app.js endSession() content.innerHTML = ` <h2>${"{"}strings.sessionComplete{"}"}</h2> <p>${"{"}strings.sessionFinished{"}"}</p> <button id="start-next-session"> ${"{"}strings.continue{"}"} </button> `;
After
Centered, gradient heading, primary CTA
3-line app.js + CSS class

Session Complete!

You completed session 1. Ready for session 2?

Gradient heading matches the progress bar. Centered layout signals "this is a milestone moment." Button is styled as a primary CTA instead of anonymous base style.
