Design Improvement #1

Button Press / Active State

Currently no visual feedback on tap. On mobile, :hover never fires — taps feel unregistered until the screen changes.

Before
No active state — buttons feel unresponsive

All buttons shown at rest (tap = nothing visible)

Primary button — tap registered? Hard to tell.
Choice button — no feedback on tap.
Quit — no tap feedback.
/* No :active rules exist */ button { transition: transform 0.15s ease, box-shadow 0.15s ease; } button:hover { transform: translateY(-2px); /* desktop only */ box-shadow: 0 6px 16px rgba(0,0,0,.25); }
After
Active state — instant tap confirmation
CSS-only fix

Buttons shown in pressed/active state

Primary button — translateY(1px) + inset shadow = pressed feel.
Choice button — background brightens on tap.
Quit — subtle background flash on tap.
button:active, a.secondary:active { transform: translateY(1px); box-shadow: inset 0 2px 4px rgba(0,0,0,.15); } .quit-button:active { background: rgba(255,255,255,0.14); transform: none; } #choices button:not(.tts-inline):active { background: rgba(255,255,255,0.28); }
